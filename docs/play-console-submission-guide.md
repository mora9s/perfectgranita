## 8. Internal testing release flow

1. Go to **Testing > Internal testing**.
2. Create or select a tester list.
3. Add the tester email addresses.
4. Create a new release.
5. Upload the production `.aab` generated by EAS.
6. Paste release notes.
7. Review warnings.
8. Roll out to internal testing.
9. Open the opt-in link on the Android phone and install via Google Play.

## 9. Pre-upload local checks

```bash
npm test
npx tsc --noEmit
node -e "const fs=require('fs'); const app=JSON.parse(fs.readFileSync('app.json')).expo; if(app.name!=='Slushi Party') throw new Error('bad app name'); if(app.android.package!=='com.mora9s.slushiparty') throw new Error('bad package'); console.log('release config ok')"
```

## 10. Production AAB build command

```bash
EXPO_NO_TELEMETRY=1 npx eas-cli@latest build --platform android --profile production --non-interactive --no-wait
```
